Core Question

Are the basics in place before you add powerful AI tools on top?

Before you worry about BAAs and HIPAA compliance, make sure your foundation is solid. These practices aren't glamorous, but they prevent the majority of security incidents.

Password Hygiene

Why This Matters

Compromised credentials are involved in nearly half of all data breaches, and healthcare has the highest average breach cost of any sector: ~$9 million per incident. If you want the full picture on why passwords are broken and what actually works, the Creating Good Passwords module goes deep. Here are the essentials.

  • Use a password manager (1Password, Bitwarden, or Dashlane). Browser-based password managers (Chrome, Safari, Firefox) have gotten much better and are fine for many people — but dedicated managers offer more flexibility, cross-platform access, and secure sharing when teams need it.
  • Every AI tool account gets a unique, randomly generated password — 16 characters minimum, mix of letters, numbers, and symbols.
  • Enable multi-factor authentication (MFA) on every AI service. No exceptions. SMS-based MFA is better than nothing, but authenticator apps (Authy, Google Authenticator) are preferred.
  • Never share login credentials. If a colleague needs access, they get their own account. Shared accounts make it impossible to audit who did what.
  • Review your AI tool accounts quarterly — disable any you're no longer using. Dormant accounts are attack surfaces.
Browser Password Managers Are Better Than Nothing

Modern browser password managers (Chrome Password Manager, Safari Keychain, Firefox Lockwise) are significantly better than reusing passwords or writing them on sticky notes. They auto-generate strong passwords and sync across devices. For many people, they're a perfectly reasonable choice.

Dedicated managers like 1Password or Bitwarden add features teams need: cross-platform vaults, secure sharing, emergency access, and breach monitoring. The best password manager is the one you'll actually use — browser-based or dedicated. The worst is none at all.

Mini-Guide: Setting Up a Password Manager

  1. Pick one: 1Password ($3/mo), Bitwarden (free tier available), or Dashlane. All work on Mac, Windows, iOS, Android.
  2. Install the browser extension and the mobile app.
  3. Create one strong master password — this is the only password you'll need to remember. Make it a passphrase: correct-horse-battery-staple style, 4+ random words.
  4. Import your existing passwords from Chrome or Safari (all managers have import tools).
  5. Enable MFA on the password manager itself.
  6. Going forward: let the manager generate and fill passwords. You'll never type a password again.

Mini-Guide: Enabling MFA on Your AI Tools

Every major AI tool supports MFA. Here's where to find it:

  • ChatGPT: Settings → Security → Multi-factor authentication
  • Claude: Settings → Security → Two-factor authentication
  • Google (Gemini): myaccount.google.com → Security → 2-Step Verification
  • GitHub (Copilot): Settings → Password and authentication → Two-factor authentication

Use an authenticator app (Authy, Google Authenticator, or your password manager's built-in TOTP) over SMS when possible. SMS is better than nothing, but SIM-swapping attacks can bypass it.

Encrypt Your Disk

The Laptop Rule

If your laptop is lost or stolen and your disk isn't encrypted, every file on it is readable. It doesn't matter how strong your password is — someone can pull the drive and read it directly.

  • macOS: FileVault — System Settings → Privacy & Security → FileVault. Turn it on. It's that simple.
  • Windows: BitLocker — Settings → Privacy & Security → Device Encryption. Enable it.
  • Chromebook: Encrypted by default.
  • iPhone/iPad: Encrypted by default when you set a passcode.
  • Android: Most modern devices encrypted by default. Verify in Settings → Security.

Verify it's on TODAY. Don't assume it is.

Mini-Guide: Checking and Enabling Disk Encryption

macOS (FileVault)

  1. Open System Settings (Apple menu → System Settings)
  2. Go to Privacy & Security
  3. Scroll down to FileVault
  4. If it says "FileVault is turned off," click Turn On
  5. Choose to save your recovery key to your iCloud account or write it down and store it somewhere safe (not on the laptop itself)
  6. Encryption happens in the background — you can keep using your Mac normally

Windows (BitLocker)

  1. Open Settings (Start → Settings)
  2. Go to Privacy & Security → Device Encryption
  3. If available, toggle Device Encryption on
  4. If you don't see this option, search for "BitLocker" in the Start menu — some Windows editions require BitLocker instead
  5. Save your recovery key to your Microsoft account or print it
Recovery Keys Matter

When you enable encryption, you'll get a recovery key. Store it somewhere safe — not on the encrypted device. If you ever forget your password or your device needs recovery, this key is the only way back in. iCloud Keychain or your Microsoft account are good choices.

Basic Computer Practices

  • Keep your OS and applications updated — security patches matter more than new features. Enable automatic updates where possible.
  • Use a VPN on public WiFi — coffee shops, airports, conference centers. Your traffic is visible to anyone on the same network without a VPN.
  • Lock your screen when you step away — macOS: Ctrl+Cmd+Q, Windows: Win+L. Make it muscle memory.
  • Don't install software from unverified sources. If you didn't seek it out from a known vendor, don't install it.
  • Be skeptical of browser extensions — especially AI ones. They can read every page you visit.
  • Enable Find My Device — macOS: System Settings → Apple Account → Find My. Windows: Settings → Privacy & Security → Find My Device.

The 5-Minute Security Audit

Right now, take 5 minutes: (1) Check FileVault/BitLocker is on, (2) Update your OS if pending, (3) Set your screen to auto-lock after 2 minutes of inactivity. These three actions prevent most physical-access attacks.
