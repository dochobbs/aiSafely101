Your AI Tools — Where the Risks Hide

AI scribes, agents, browser extensions, developer tools, and the nine categories of tools that catch people off guard.

~15 min read 6 readings
Core Question

You've vetted the big tools. But what about the extensions, automations, and "helpful" add-ons running in the background?

AI Agents — When the Tool Starts Doing Things on Its Own

Most AI tools you've used so far are conversational — you type, they respond. AI agents are different. You give them a goal and they go do things: browse websites, write files, run code, send messages, install software. Think of the difference between asking a colleague a question and handing them your laptop with your passwords.

The OpenClaw Incident

In early 2026, an AI agent called OpenClaw went viral — evolving from Clawdbot to Moltbot to OpenClaw in just five weeks. It exposed a critical truth: autonomous AI agents can access your API keys, credentials, and any data they can reach. When you give an agent permission to act on your behalf, you're trusting it with everything it can touch.

Compare this to Anthropic's managed agentic features — computer use, tool use, and the Agent SDK — which operate within defined permission boundaries. Managed tools have explicit permission systems; unmanaged agents often don't.

Claude Code, MCP Servers, and Developer Tools

Who Needs This Section

This section is for people who use or manage developer tools — or who manage people who do. If you're clinical staff and terms like "npm" and "API keys" aren't part of your day, here's the one takeaway: don't install developer tools, browser extensions, or software plugins that your organization hasn't approved. Then skip ahead to Nine More Tools That Catch People Off Guard.

Claude Code is a powerful developer tool. It can read your files, execute commands, and interact with external services through MCP (Model Context Protocol) servers.

Real Vulnerabilities, Real CVEs

In early 2026, Check Point Research discovered critical vulnerabilities in Claude Code (CVE-2025-59536, CVE-2026-21852) that allowed remote code execution and API key exfiltration through malicious project configurations.

Between January and February 2026, researchers filed over 30 CVEs targeting MCP servers. Among 2,614 MCP implementations surveyed, 82% were vulnerable to path traversal attacks.

  • MCP servers get broad access to your filesystem, APIs, and credentials. Only install MCP servers from trusted, audited sources.
  • Code and context are sent to Anthropic's servers for processing. Don't open projects containing PHI in Claude Code.
  • API keys can be exfiltrated if you clone and open untrusted repositories. Review .claude/ directories before trusting a project.
  • npm/pip packages suggested by the tool may be typosquatted or malicious. Verify packages before installing.

The Dependency Problem

Here's something that catches even experienced developers off guard: Claude Code will freely suggest installing packages and dependencies as part of its workflow. When you ask it to build something, it will often run npm install some-package or pip install some-library without hesitation. Each of these commands pulls code from public registries — and that code runs on your machine with your permissions.

This matters because:

  • Typosquatting is real: Malicious packages with names like reqeusts (misspelling of requests) exist on npm and PyPI specifically to catch careless installs. Claude Code might suggest the wrong package name.
  • Dependencies have dependencies: Installing one package can pull in dozens of transitive dependencies, each with its own code that executes on your machine.
  • Post-install scripts run automatically: npm packages can include scripts that execute immediately upon installation — before you've reviewed any code.
  • Your environment is the blast radius: Whatever permissions you have — access to files, environment variables, API keys, network — the installed package has too.
What to Do About Dependencies

When Claude Code suggests installing a package:

  • Check if the package name is spelled correctly and matches what you expect
  • Verify the package exists on npmjs.com or pypi.org — check download counts, last publish date, and maintainer
  • Be especially wary of packages with very low download counts or recent publish dates
  • Never run Claude Code in --dangerously-skip-permissions mode on machines with access to sensitive data
  • Review .claude/ directories in repositories before trusting them

This isn't about being paranoid — it's about being as thoughtful with AI-suggested code as you would be with AI-generated clinical recommendations. Trust but verify.

Be Mindful Using Corporate AI for Personal Tasks

If your organization provides you with a ChatGPT Enterprise account, a Claude Enterprise workspace, or a Copilot license — it's tempting to use it for personal tasks too. Drafting a personal email. Planning a vacation. Asking about a medical question for yourself or a family member.

This isn't primarily about "breaking the rules" — it's about understanding that corporate AI tools aren't like Google:

  • Memory and context: Many enterprise AI tools have memory features that persist between conversations. Your personal query about a family member's health condition may surface in a future work conversation.
  • Saved conversations: Your organization can likely see your conversation history. What feels like a private chat is visible to IT administrators.
  • Usage analytics: Organizations track how tools are used. Your personal usage is part of that data.
  • Training and improvement: Depending on the organization's agreement, conversations may be used to improve the tool — your personal data included.
The Simple Rule

Use your organization's AI tools for work. Use your personal accounts for personal tasks. Keep the streams separate — not because you'll get in trouble, but because the tools are designed to remember, learn, and surface context. You don't want your weekend recipe search showing up in a Monday morning patient note suggestion.

Browser AI Extensions

AI browser extensions (Monica, Merlin, Sider, and dozens of others) can read every webpage you visit — including your open EHR session, your email, and your banking portal. Most have no BAA.

The Extension Rule

If an AI browser extension can "read and change all your data on all websites," it can see your patient data when you have your EHR open. Treat browser extensions like you'd treat giving someone your login credentials — because functionally, that's what you're doing.

"I ship code I don't read." — Peter Steinberger, OpenClaw creator, who made 6,600 commits in January 2026 alone, almost all generated by AI agents.
The Bottom Line on AI Agents

An AI agent can only do what you've given it permission to do — but most people don't realize how much permission they've given. If it can read your files, it can read all your files. If it can run commands, it can run any command. Ask yourself: would I hand my unlocked laptop to a stranger and say "do whatever you think is best"? That's what running an unvetted agent does.

Nine More Tools That Catch People Off Guard

1

Zapier/Make/n8n Automations — Low-code tools that move data between services. If you're automating anything involving patient data, every service in the chain needs a BAA.

2

AI PDF and Document Tools — Uploading patient documents to AI summarizers, OCR tools, or "smart" PDF readers. If the tool processes documents on external servers, you need a BAA.

3

Voice and Transcription Apps — Otter.ai, meeting bots, Whisper-based tools that auto-record. Remember the Ontario Hospital breach — a bot joined a meeting, recorded seven patients' data, and emailed it to 65 people.

4

Code Assistants (Copilot, Cursor, Windsurf) — These transmit your code to external servers. If your code contains patient data — even in comments or test fixtures — that's a potential HIPAA violation. Most do not offer BAAs.

5

Local Model Downloads — Pulling models from Hugging Face or similar. Models can contain malware or phone home. Verify the source and scan before running.

6

AI Image/Video Tools — Uploading photos or screenshots for AI processing. If those images contain patient information, you've just transmitted PHI to an uncovered service.

7

"Smart" Note-Taking Apps — Notion AI, Obsidian plugins with AI features. Check whether AI features transmit data to external servers and whether BAAs are available.

8

Email AI Features — Gmail's "Help me write," Outlook Copilot. These may or may not be covered under your organization's BAA. Verify before using with patient communications.

9

AI-Powered Search — Perplexity, Arc browser with AI features. Your queries may be stored and associated with your account. Don't search for patient-specific information on uncovered platforms.
